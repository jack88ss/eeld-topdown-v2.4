# ⚡ 自动决策功能 - 快速参考

## 🎯 一句话总结

**超过 1 分钟用户未确认时，Sisyphus 自动评估并决策是否继续。**

---

## ⏱️ 工作原理

```
用户确认环节 → 等待 60 秒 → 超时 → 自动评估 (0-100分) → 智能决策
```

---

## 🎯 决策规则

| 得分范围 | 决策 | 说明 |
|---------|------|------|
| **≥ 90 分** | ✅ 直接继续 | 提案优秀，无需修改 |
| **80-89 分** | 🔧 微调后继续 | 提案良好，小幅优化 |
| **< 80 分** | 📝 补充备选方案后继续 | 需改进，提供 2-3 个替代方案 |

---

## 📊 评估标准

| 维度 | 权重 | 说明 |
|------|------|------|
| 教学目标对齐度 | 30% | 是否达成课程目标 |
| 学生能力匹配度 | 30% | 难度是否适合学生 |
| 任务可行性 | 25% | 课堂实施是否可行 |
| 创新性与趣味性 | 15% | 是否激发学生兴趣 |

---

## 📝 决策记录

所有自动决策都记录在：

```
state/DECISION_LOG.md
```

包含：
- ⏰ 决策时间
- 📊 评估得分
- 🎯 决策类型
- 📝 调整内容
- 💡 决策理由

---

## 🔧 自定义设置

修改 `.opencode/oh-my-opencode.config.json`：

```json
{
  "agents": {
    "sisyphus": {
      "autonomy": {
        "decision_making": {
          "user_timeout_threshold_seconds": 60,  // 改为 30 或 120
          "quality_threshold": 80                // 改为 70 或 90
        }
      }
    }
  }
}
```

## 💡 使用场景

### ✅ 适合

- 🌙 **批量生成**：睡前启动，早上查看结果
- 🏃 **无暇确认**：临时离开，无需暂停工作流
- 🤖 **信任 AI**：相信 Sisyphus 的专业判断

### ⚠️ 不适合

- 🎯 **特殊要求**：有非常具体的大任务想法
- 👀 **实时参与**：想要实时查看和调整每个环节
- 📚 **首次使用**：建议第一次手动确认，了解系统

---

## 🎉 优势

| 优势 | 说明 |
|------|------|
| ⏰ **无需值守** | 启动后可以离开，自动完成 |
| 🧠 **智能决策** | 基于多维度评估，不是简单通过 |
| 📈 **批量生成** | 一次生成多个单元教案 |
| 📝 **全程记录** | 所有决策可追溯，透明可信 |
| 🎯 **质量保证** | 低分方案会补充备选，确保质量 |

## 🚀 开始使用

1. 启动 OpenCode：
   ```bash
   cd C:\Users\Administrator\Desktop\myproject\eeld-top-down
   opencode
   ```

2. 输入任务（可以离开了）：
   ```
   ulw Design lesson: Unit 1 Goldilocks, 五年级, 译林版
   ```

3. 等待完成（60 秒超时会自动决策）

4. 查看结果：
   ```
   draft/lesson_plan.md          # 完整教案
   state/DECISION_LOG.md         # 决策记录
   ```
