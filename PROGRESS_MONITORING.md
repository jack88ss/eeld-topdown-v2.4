## 🔍 判断是否卡住

### 正常情况

- ✅ 进度百分比持续增长
- ✅ 最近更新时间在 5 分钟以内
- ✅ state/ 目录下有新文件生成

### 异常情况（可能卡住）

- ⚠️ 进度超过 **10 分钟**没有变化
- ⚠️ 最近更新时间超过 **15 分钟前**
- ⚠️ OpenCode 窗口没有任何输出

---

## 🛠️ 卡住了怎么办

### Step 1: 检查 OpenCode 状态

打开 OpenCode 窗口，查看：
- 是否在等待用户输入？
- 是否显示错误信息？
- 是否还在运行？

---

### Step 2: 查看日志

```powershell
# 查看最新日志
type state\LOG.md | Select-Object -Last 20

# 查看 OpenCode 输出
# (在 OpenCode 窗口查看)
```

---

### Step 3: 手动继续

如果确认卡住，可以：

**选项 1：重新启动工作流**
```powershell
# 按 Ctrl+C 停止当前 OpenCode
# 重新运行
opencode
# 输入: ulw Design lesson: [任务描述]
```

**选项 2：从卡住的阶段继续**
```powershell
# 假设卡在阶段 4，手动生成后续阶段
# (联系 AI 助手帮助完成剩余阶段)
```

## 📱 自动监控（进阶）

### 定时检查脚本

创建 `monitor.bat`：
```batch
@echo off
:loop
cls
call .opencode\check-progress.bat
timeout /t 60 /nobreak
goto loop
```

**运行**：
```
monitor.bat
```

每 60 秒自动刷新一次进度，方便长时间监控。

## 💡 最佳实践

### 启动工作流后

1. **立即运行一次进度检查**
   ```
   .\.opencode\check-progress.ps1
   ```
   确认阶段 1 已开始

2. **5-10 分钟后再检查一次**
   看是否进展到阶段 2-3

3. **如果需要离开**
   - 记下当前进度百分比
   - 回来后再检查，对比是否增长

4. **阶段 4 会暂停等待确认**
   - 如果 60 秒内没确认，Sisyphus 会自动决策
   - 可以提前准备好要不要修改大任务
